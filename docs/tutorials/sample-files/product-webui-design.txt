Section 1: Navigation Systems

The new website design is discussed from the perspective of navigation
systems. The document repeatedly refers to localisation, localized user
interfaces, language support, website design, interface consistency,
translation workflows, locale management, internationalisation,
multilingual content, and the new website redesign. This section
intentionally explores localisation in depth without reporting any
implementation status, completion percentage, milestone, meeting
outcome, approval, decision, or release readiness. Instead it describes
general principles, recommendations, architectural considerations,
editorial guidelines, user expectations, design patterns, review
practices, terminology management, content governance, and long-term
planning. When discussing localisation of the new website design, the
emphasis is always on possibilities, requirements, examples, and best
practices rather than actual progress. Examples include translation
workflows, language fallback, locale-aware formatting, accessibility,
consistent navigation labels, multilingual search, content
synchronization, and reusable UI components. Readers should understand
that this material is descriptive background intended for designers and
developers. It deliberately avoids documenting the current state of
localisation of the new website design.
